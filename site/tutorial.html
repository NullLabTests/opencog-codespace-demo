OpenCog AtomSpace Tutorial

Interactive guide to the Atomese language. Paste these commands into the WebSocket Shell or pipe with nc.

Step 1: Create Concepts

Atoms are the building blocks. Concept atoms represent named entities.

(Concept "apple")
(Concept "fruit")
(Concept "red")

Step 2: Build Relationships

Links connect concepts into a knowledge graph.

; apple IS-A fruit
(Inheritance (Concept "apple") (Concept "fruit"))

; apple HAS-PROPERTY red
(Property (Concept "apple") (Concept "red"))

Step 3: Query the Graph

Use Get with variables to find matching patterns.

; Find everything that inherits from "fruit"
(cog-execute!
    (Get (Inheritance (Variable "$what") (Concept "fruit"))))

Step 4: Graph Rewriting

Bind finds a pattern and creates new atoms from it.

; Everything that IS-A fruit also IS-A food
(cog-execute!
    (Bind
        (Inheritance (Variable "$x") (Concept "fruit"))
        (Inheritance (Variable "$x") (Concept "food"))))

Step 5: Arithmetic

Atomese has built-in arithmetic evaluation.

(cog-evaluate! (Plus (Number 3) (Number 4)))
(cog-evaluate! (Times (Number 6) (Number 7)))
(cog-evaluate! (GreaterThan (Number 10) (Number 5)))

Step 6: Truth Values

Atoms can carry probabilistic truth values (strength, confidence).

; Create with 90% strength, 80% confidence
(cog-new-link 'InheritanceLink
    (Concept "cat") (Concept "animal")
    (cog-new-stv 0.9 0.8))

Step 7: Knowledge Patterns

Common real-world patterns in Atomese.

; Set membership
(Member (Concept "alice") (Concept "programmers"))

; Event at a time
(AtTime (Concept "meeting") (Time "2026-06-01T17:00:00"))

; If-then rule
(Implication (Concept "rain") (Concept "wet-ground"))
